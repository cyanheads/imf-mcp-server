# 0.2.1 — 2026-06-12

## Added

- **`scripts/clean-mcpb.ts`**: post-pack MCPB cleaner wired into the `bundle` script. Runs `mcpb clean` (dev-dependency prune) then strips dependency-shipped agent docs nested under `node_modules/` (`skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach, and asserts zero remain.
- **`lint-packaging` bundle-content and identity guards**: new checks reject unanchored `.mcpbignore` dev-dir patterns that would also strip nested runtime paths, verify a built `.mcpb` carries no `node_modules/**` agent-doc entries, and require the `createApp()` `name`/`title` literals and manifest `display_name` to equal the unscoped package name.
- **`check-framework-antipatterns` `z.coerce.boolean()` rule**: flags the env-boolean footgun (`Boolean("false") === true`) in `src/` config and recommends `z.stringbool()`; comment lines are skipped so docs naming the pattern don't trip it.

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.21` → `^0.10.6`.
- **`imf_list_databases` `notice`**: moved from the output schema to the framework `enrichment` block, populated via `ctx.enrich.notice()`. `format()` no longer renders it from the domain payload — the framework mirrors the enrichment into the `content[]` trailer.
- **`imf_dataframe_query` / `imf_query_dataset` error codes**: `invalid_sql` and `key_dimension_mismatch` now classify as `ValidationError` instead of `InvalidParams`.
- **`imf_dataframe_query` canvas query**: passes `denySystemCatalogs: true` to block SQL access to DuckDB system catalogs.
- **`createApp()` identity**: explicit `name` and `title` set to `imf-mcp-server`.
- **`bundle` script**: builds with `bun run build` and runs `scripts/clean-mcpb.ts` after `mcpb pack`.
- **`.mcpbignore`**: dev-dir patterns root-anchored with a leading `/` so they no longer match nested `node_modules/**` runtime paths.
- **`Dockerfile`**: `APP_VERSION` build arg drives an `org.opencontainers.image.version` label; adds a writable `.cache` dir for the on-disk DuckDB canvas store and a bun-native `HEALTHCHECK` against `/healthz`.
- **`.claude-plugin/plugin.json` / `.codex-plugin/mcp.json`**: `npx` argument corrected to the scoped package name `@cyanheads/imf-mcp-server`.
- **`@biomejs/biome`** `^2.4.16` → `^2.5.0`.
