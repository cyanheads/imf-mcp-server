# 0.2.0 — 2026-06-10

## Added

- **`imf_get_database` `codelist_filter`**: new optional input — case-insensitive substring over code ID and name; when set, returns all matching entries per dimension with no cap (bypasses the 50-entry ceiling). Truncation notice in `format()` now names the escape hatch: `codelist_filter` and `imf://database/{dataflow_id}`. ([#8](https://github.com/cyanheads/imf-mcp-server/issues/8))
- **`imf_query_dataset` availability enrichment**: on empty result, queries the SDMX 2.1 `availableconstraint` endpoint before throwing `no_data`. `series_count=0` → error states the code has no coverage in this dataflow. `series_count>0` → error includes per-dimension available codes and time range from the cube region. Availability fetch failure degrades to the existing generic `no_data` — never masks the original error. ([#5](https://github.com/cyanheads/imf-mcp-server/issues/5))
- **`AvailabilityResult` type** added to `src/services/imf-sdmx/types.ts`: `series_count`, `available_codes`, `time_period_start`, `time_period_end`.
- **`ImfSdmxService.fetchAvailabilityConstraint()`**: queries `…/sdmx/2.1/availableconstraint/{dataflow}/{firstCode}..`; parses `series_count` annotation, cube-region `com:KeyValue` entries (capped at 20 per dimension), and time annotations from the XML response.

## Changed

- **`imf_query_dataset` period filtering**: `start_period`/`end_period` now filter observations server-side after `fetchData` returns, before canvas spill. Handles upstream monthly format `YYYY-MNN` (e.g. `1956-M01`) against `YYYY-MM` input via a `normalizePeriod` helper. `format()` period line now reads `**Period:** <range>` — the "full available series" caveat is removed. ([#6](https://github.com/cyanheads/imf-mcp-server/issues/6))
- **`imf_query_dataset` null-padding removal**: observations with both `value=null` and `status=null` are dropped before the inline/spill split. Rows with a non-null `status` (official missing-value markers) are kept. A result all-null after filtering falls through to `no_data`. `observation_count` reflects post-filter count. ([#7](https://github.com/cyanheads/imf-mcp-server/issues/7))
- **`imf_list_databases` `filter` description**: corrected from "name or ID substring" to "name, ID, or description substring" — the filter already searches descriptions, only the description was wrong. ([#9](https://github.com/cyanheads/imf-mcp-server/issues/9))
- **`imf_dataframe_describe` / `imf_dataframe_query` `canvas_not_found` recovery**: removed the `CANVAS_PROVIDER_TYPE=duckdb` env-var hint, which is ops-level context the client cannot act on. Recovery now reads: "Re-run imf_query_dataset to obtain a fresh canvas_id." ([#9](https://github.com/cyanheads/imf-mcp-server/issues/9))
- **`imf_get_database` / `imf_query_dataset` descriptions**: added a note that codelists enumerate the code universe, not actual coverage — valid codes can still return `no_data` if the combination has no series.
- **`@types/node`** `^25.9.2` → `^25.9.3`; removed from `devcheck.config.json` outdated allowlist.
